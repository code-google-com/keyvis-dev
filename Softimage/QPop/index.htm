QPop for Softimage

Stefan Kubicek & Eugen Sares


QPop is a custom quad-menu system for Softimage similar to 3dsMax's quad menu system. The goal is to create a quad menu system that can be configured in a user friendly way. That is: through clicking only.


To keep QPop as flexible as possible, several aspects of QPop are script based so that is easily extendable without having to change the actual plugin source code.

When QPop will be released, a large enough set of existing configuration scripts should exist so that the casual user can use it without ever having to script anything. At that point it shall provide the same ease of use as 3dsMax's quad menu system.



Features Overview


Qpop Views

Each window and view in Softimage can have it's own menu set (s) (see below on menu sets). Theoretically an unlimited amount of menu sets can be configured for any view. In practice, since menu sets can be context sensitive (e.g. sensitive to what is currently selected, or pretty much anything else that can be queried through scruipting in any way), we assume that four (4) menu sets per view should be enough. This number can be increased easily whenever needed.

Menu set invocation is command based – Currently we have four commands implemented to display menu set 0 -3 per view (QpopDisplayMenuSet_1 to QpopDisplayMenuSet_3, assignable to hot keys)


QPop Menu Sets

A menu set can contain any number of menus, of which alyways a maximum of four main menus (besides their submenus ) are displayed. Which menus actually get displayed can be configured using display contexts (see below on display contexts). In above figure a menu set displaying 2 menus is visible, the other two slots are not in use because no menu was assigned to them in the current context and view.


QPop Display Contexts

These are user-editable scripts assigned to menu sets which evaluate to True or False. All scripting languages are supported.

For example, a display context script may evaluate to True if the selected object is a polygon point, as illustrated in avove figure. The menu associated with the context is configured so it contains only operations suitable for point editing.

If there are more than one display contexts evaluating to True for the current view, the menu associated with the first display context evaluating to True will be displayed. Therefore,

the order in which display contexts are defined for a given view does matter. This order can be changed at any time.



QPop Menus

A Qpop menu contains the actual menu items a user can click.

Menu items can be:

  • Softimage commands (native or custom)

  • Other Qpop menus, which are displayed as sub-menus (but no native Softimage menus (yet?))

  • Scripted menu items (all scripting languages supported)

Qpop menus can contain fixed menu items that are saved to the config file, and temporary menu items which do not get saved. A menu may have associated script code (all scripting languages supported) which gets executed before the menu is displayed. This is useful to fill in menu content on the fly depending on context or certain conditions. For example, a menu could display a list of existing lights in the scene for quick selection. Since such menu items are only meaningful in the current scene, they can be stored int the menu's temporary menu items list so they don't get saved in the current configuration file by accident once it is saved.


Command repetition

To repeat the last operation performed by Qpop the command „QpopRepeatLastCommand“ can be bound to a hotkey. Alternatively, the last command can be repeated by clicking on one of the quad menus's menu titles. Currently, the last executed Qpop menu item is repeated independant of the clicked menu title the user clicks. In future versions, this could be changed so that the last executed menu item of the respective menu's title upon which the user has clicked, is executed (3dsMax-style).


Saveable configuration

At any time the current Qpop configuration can be saved to an xml file.

This file (or any other config file ) can be specified in preferences to be automatically loaded next time Softimage starts.

An import feature with duplicates and name clash handling is planned to make it easier to manage and share config files them, with other users.



Installation

0.)

Exit Softimage

1.)

Make sure Python with pywin and Mocrosoft's .NET 3.5 framework are installed on your machine. (Qpop menu rendering makes use of .Net, the Qpop Configurator is written in Python and uses some pywin functions)

2.)

Download the xsi addon Qpop & QpopConfigurator (right-click to save linked content as...) and install it, e.g. through drag & drop into XSI.

3.)

Start Softimage and, click File->Preferences...->Custom->QPop to open the QpopConfigurator PPG.

Note: We've seen display problems with complex PPG's displayed inside the Preferences window.

If the QPop PPG looks corrupted after having been resized, click on the orange custom preference icon left of the Qpop entry to open the Qpop configurator in a separate property page independant of the preferences window. But be careful not to open 2 instances of the qpop preferences, neither separate nor inside another preferences window. This would cause an update cycle. This is currently a limitation.

Alternatively, resizing the right side pane of the preferences window by slightly moving the thin separator on the left also results in correct rendering of the PPG. This needs to be done every time the PPG has been resized, but seems to happen only on 64 bit systems. Since we are not doing anything unusual in the PPG script we believe this is a Softimage problem (bug?) that we can't do anything about. Please let me know if you know of a solution to this problem (stefan@tidbit-images.com)


4.)

Optional, but recommened for a start: Set the Qpop Configuration file path to the provided demo configuration file at

C:\users\<YourUserName>\Autodesk\Softimage_7.5\Addons\QPop\Data\Preferences\QPopConfiguration_Default.xml

and load the file by pressing the „Load“ button left of the file path. It file contains some demo menus and contexts defined for the main 3D Views.


Update: Qpop should find the default config xml at first startup automatically now. Please let me know if it does not (stefan@tidbit-images.com).


5.)

Assign the QPopDisplayMenuSet_0 command to a hotkey of your choice. You could do the same for QpopDisplayMenuSet_1 – 3, but it is unlikely you will need them yet.



Testing your Qpop installation


If you had no errors so far it is very likely that Qpop will work.

To do a quick test, create any polymesh object, move your mouse over one of the 3D views and press the hotkey associated with the QpopDisplayMenuSet_0 command.

A menu should come up that looks smilar to the one visible in the illustration on the top of this page.

Select one or more points on the mesh, and invoke the Qpop menu as before. A different menu showing some commands relevant to point editing should become visible.


If you encounter any errors at any stage of installation or testing, please write an email to stefan@tidbit-images.com

Technical details


WIP


QPop consists of two components

  1. Qpop.dll C++ plugin by Eugen Sares, compiled for both Softimage 7.5 – 32 and 64 bit.

A version for Softimage 7.0 will also be available later.

This plugin implements the Qpop command, which renders the menu set described by a string. It returns the menu number and item number clicked by the user.


  1. The QpopConfigurator.py Python plugin by Stefan Kubicek, which allows for menu configuration and deals with interpreting and executing menu items etc. It embeds the configurator as a PPG in custom preferences.
